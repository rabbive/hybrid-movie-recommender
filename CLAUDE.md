# CLAUDE.md — AI Assistant Guide for hybrid-movie-recommender

## Project Overview

A hybrid movie recommendation system combining **TF-IDF content-based filtering** and **SVD collaborative filtering**. Built as an academic project with a Streamlit web UI. The codebase is intentionally small (~450 lines across 4 Python files).

## Development Workflows

### Initial setup

```bash
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### Training (required before running the app)

```bash
python train.py
```

This calls `recommender.train_all()`, which:
1. Loads and merges the 4 CSV files
2. Parses JSON-like strings in genres/keywords/cast/crew with `ast.literal_eval()`
3. Builds a TF-IDF matrix on concatenated `genres + keywords + cast + director + overview`
4. Computes the full cosine similarity matrix (dense, written to `cosine_sim.pkl` — 93 MB)
5. Runs 3-fold cross-validation then a final 80/20 train-test split with SVD
6. Saves 8 pickle files to the repo root
7. Runs a sanity-check recommendation (`avatar`, user 1) before saving

Training produces console output:
```
CV RMSE (mean ± std): X.XXXX ± X.XXXX
Holdout test RMSE: X.XXXX
✅ All models saved successfully!
```

### Running the Streamlit app (standalone, no API needed)

```bash
streamlit run app.py
```

All pickle files must exist before launching. The app loads them once via `@st.cache_resource` and keeps them in memory for the session.

### Running the Next.js frontend + FastAPI backend

Terminal 1 — start the API (must run from repo root so pickles are found):
```bash
uvicorn api:app --reload --port 8000
```

Terminal 2 — start the Next.js dev server:
```bash
cd frontend
npm install   # first time only
npm run dev
```

Open `http://localhost:3000`. The frontend calls `http://localhost:8000`.

**CORS**: `api.py` allows `http://localhost:3000` only. For other origins update `allow_origins` in `api.py`.

### No test suite

There are no automated tests. The only validation is the sanity-check recommendation run at the end of `train_all()`. Manual testing is done by:
1. Running `python train.py` and confirming it completes without error
2. Launching `streamlit run app.py` and exercising the UI

## Architecture: How the Three Core Files Interact

```
train.py
   └── recommender.train_all()         ← training only
         ├── [build TF-IDF, cosine_sim, indices]
         ├── [train SVD model via Surprise]
         ├── recommender_core.hybrid_recommend()  ← sanity check
         └── [pickle 8 artifacts]

app.py
   ├── load_models()  [@st.cache_resource]  ← loads pickles
   └── run_recommend()
         └── recommender_core.hybrid_recommend()  ← called on every button press
```

`recommender_core.py` is the **only shared module**. Never put training logic in it and never put UI logic in it.

## Core Algorithm: `hybrid_recommend()` (recommender_core.py)

**Signature:**
```python
hybrid_recommend(
    movies, cosine_sim, indices, model, ratings,
    user_id, movie_title,
    top_n=5, neighborhood=19,
    cf_weight=0.6, content_weight=0.4
) -> pd.DataFrame | None
```

**Logic:**
1. Lowercase `movie_title`, look it up in `indices`. Return `None` if not found.
2. Determine if `user_id` is in `ratings["userId"]` (warm vs cold-start).
3. Retrieve the top `neighborhood` (default 19) cosine neighbors of the seed movie.
4. **Warm user path**: For each neighbor, call `model.predict(user_id, movie_id).est` (SVD). Min-max normalize both predicted ratings and similarities within the neighborhood pool, combine as `0.6 × pred_norm + 0.4 × sim_norm`.
5. **Cold-start path**: `final_score = cosine similarity` (SVD is skipped entirely).
6. Sort by `final_score` descending, return top `top_n` rows.

**Returned DataFrame columns:** `title`, `movieId`, `similarity`, `predicted_rating` (None for cold-start), `pred_norm`, `sim_norm`, `final_score`.

**Key invariant:** Movie titles are stored and looked up in **lowercase** throughout the pipeline. `indices` keys are lowercase; `movies["title"]` values are lowercase. The UI converts to title case only for display.

## Data Flow and Key Conventions

### Feature engineering (recommender.py)
- `convert(text)`: parses a JSON string → list of names with spaces removed (e.g., `"Science Fiction"` → `"ScienceFiction"`)
- `get_director(text)`: scans crew JSON for first entry with `job == "Director"`
- Cast is truncated to top 3 members before joining
- Final `features` column: `genres + " " + keywords + " " + cast + " " + director + " " + overview` (all lowercase)

### ID alignment
- TMDB uses `movie_id` (from tmdb_5000_movies.csv)
- MovieLens uses `movieId`
- `links_small.csv` bridges them via `tmdbId`
- After the merge, `movies` DataFrame has both `movie_id` (TMDB) and `movieId` (MovieLens)
- The SVD model uses `movieId` (MovieLens IDs); cosine similarity uses DataFrame row indices

### Pickle file dependency order
When regenerating, all 8 pickles must come from the **same training run**. Mixing pickles from different runs will produce silently incorrect results because row indices in `cosine_sim` correspond to the specific order of `movies` from that run.

## Streamlit App Conventions (app.py)

- `load_models()` is decorated with `@st.cache_resource` — it runs only once per process. Do not add mutable state here.
- Recommendation results are stored in `st.session_state.results` so they persist across re-renders.
- `dataset_stats.pkl` and `cv_metrics.pkl` are loaded with `try/except OSError` — the app degrades gracefully if they are missing.
- All charts use `plt.close(fig)` after `st.pyplot()` to prevent memory leaks.
- Movie titles are lowercased everywhere internally; `.str.title()` is applied only in display DataFrames and chart labels.

## Important Constraints and Gotchas

1. **Python version**: Use 3.10, 3.11, or 3.12. scikit-surprise does not build on 3.13+.
2. **NumPy < 2**: `numpy>=1.24,<2` is pinned. Do not upgrade beyond 1.x — scikit-surprise's Cython extensions break.
3. **Working directory**: Both `recommender.py` and `app.py` read CSV/pickle files using bare filenames (e.g., `pd.read_csv("ratings_small.csv")`). Always run commands from the repo root.
4. **cosine_sim.pkl is 93 MB**: Loading it is slow on first run; `@st.cache_resource` ensures it only loads once. Do not attempt to load it multiple times in the app.
5. **No environment variables / config files**: Hyperparameters (`cf_weight=0.6`, `content_weight=0.4`, `neighborhood=19`, `top_n=5`) are hardcoded as function defaults. Change them at the call site in `app.py:run_recommend()` or `recommender.py` sanity check.
6. **Pickle compatibility**: The pickle files were produced with a specific Python + library version. After any `pip install --upgrade`, retrain by running `python train.py` to regenerate all pickles.
7. **No CI/CD**: There is no automated pipeline. All validation is manual.
8. **Large files committed**: The `.gitignore` does not exclude CSV data or pickle artifacts. This is intentional — the repo ships with precomputed models.

## Common Tasks

### Add a new movie metadata feature
1. Edit `recommender.py` — modify the `features` column construction (lines 62–73).
2. Re-run `python train.py` to regenerate all pickles.
3. No changes needed in `recommender_core.py` or `app.py` — the feature vector is opaque to them.

### Change hybrid weights
- Edit the `cf_weight` / `content_weight` arguments at the `hybrid_recommend()` call site in `app.py:run_recommend()` (line 91–100).
- The defaults in `recommender_core.py` are fallbacks; prefer explicit values at the call site.

### Add a new visualization
- Add a new `st.checkbox` block in `app.py` under "Figures for analysis" (after line 131).
- Follow the existing pattern: create a `fig, ax`, plot, call `st.pyplot(fig)`, call `plt.close(fig)`.

### Retrain the model
```bash
python train.py
# Outputs CV RMSE and holdout RMSE, then saves 8 pickle files
```

### Change the number of recommendations
- Default is `top_n=5`. Change the argument in `app.py:run_recommend()`.
